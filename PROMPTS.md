# PROMPTS.md — Claude Code 세션별 착수 프롬프트

각 마일스톤 시작 시 아래 프롬프트를 그대로 채팅에 붙여넣는다.
CLAUDE.md는 자동으로 읽히므로 여기서는 "이번에 뭘 하는지 + 완료 확인 기준"만 지시한다.

---

## M1 — 코어 루프

```
CLAUDE.md를 읽고 M1을 시작해.

이번 세션 산출물:
1. 파일 구조 전체 생성 (§4-2). lang/en.js, ja.js는 빈 골격만.
2. config.js에 §7-2 표의 1~2단계 파라미터 정의
3. 스폰 시스템: 해바라기/시든 꽃이 §7-2 규칙대로 등장/소멸
4. 탭 판정: 해바라기 = 포인트 획득 + 플로팅 텍스트, 시든 꽃 = 5% 감소 + 화면 흔들림
5. 진화 1회 (씨앗 → 새싹, 50포인트, 선택지 없이 자동)
6. i18n 골격 + ko.js (이 세션에서 쓰는 텍스트만)

완료 확인: index.html을 브라우저로 열어서 60초 플레이가 가능하고,
포인트가 오르내리고, 50포인트에서 새싹으로 진화하면 M1 완료.
시작 전에 질문 있으면 가장 중요한 것 1~2개만 물어봐.
```

---

## M2 — 진화 트리 전체

```
M1이 완료됐다. M2를 시작해.

이번 세션 산출물:
1. §7-3 진화 6단계 전체 + 각 단계 선택지 모달 UI (A/B 카드 선택)
2. §7-2 표의 3~6단계 파라미터를 config.js에 추가, 진화 시 적용
3. 콤보 시스템 (§7-2): 노 미스 10회마다 ×1.1, 실수 시 리셋, HUD에 표시
4. 히든 태양형: 노 미스로 4단계 도달 시 세 번째 선택지 등장. 조건은 UI에 노출 금지
5. localStorage 자동 저장/로드 (진행 상황 전체)

완료 확인: 새로고침해도 이어서 플레이되고, 노 미스 플레이로 태양형
선택지가 뜨는 것 확인. 테스트 편의를 위해 URL 파라미터 ?debug=1일 때
포인트 치트(+1000 버튼)를 추가해 — 배포 빌드에서 제거하지 말고 그대로 둬도 됨
(이스터에그 게임의 이스터에그로 취급).
```

## M3 — 보스전 + 엔딩 (1차 완결)

```
M2가 완료됐다. M3를 시작해.

이번 세션 산출물:
1. §7-5 보스전: 최종형 도달 시 보스전 진입 버튼 → 태양 등장 연출 →
   해바라기만 탭해서 데미지, 시든 꽃 탭 시 보스 HP 2% 회복
2. 일반 엔딩: 보스 격파 → §7-6 통계 화면
3. 태양형 진 엔딩: 보스전 대신 대화 씬(탭으로 대사 넘김) → 전용 통계 화면
4. 엔딩 후 "처음부터" 버튼 (localStorage 초기화)

대사는 ko.js에 초안으로 작성해 (톤: §7-4). 내가 검토 후 수정 지시할게.

완료 확인: debug 치트로 최종형까지 간 뒤 두 엔딩 모두 도달 가능하면
M3 완료. 이 시점에서 게임은 "완결 가능" 상태여야 한다.
```

## M4 — 유머 텍스트 + 3개국어

```
M3가 완료됐다. M4를 시작해.

이번 세션 산출물:
1. §7-4 유머 레이어 완성: 혼잣말 6개(50/100/200/400/700/1000회),
   진화 코멘트 6개, 보스/진엔딩 대사 — 전부 ko.js에
2. en.js, ja.js 전체 번역. §8-5 톤 규칙 준수 (직역 금지, 언어별 유머 현지화)
3. 언어 전환 UI: 시작 화면 언어 선택 + 인게임 우상단 🌐 버튼, 즉시 반영
4. i18n.js에 누락 키 감지: 세 파일의 키 셋을 비교해서 불일치 시 콘솔 경고

완료 확인: 세 언어 모두로 시작→엔딩까지 텍스트 깨짐/누락 없이 표시.
일본어는 이후 원어민 검수가 있을 예정이니, 검수자가 보기 쉽게
ja.js에 키별로 한국어 원문을 주석으로 병기해 줘.
```

## M5 — 폴리싱 + 밸런싱

```
M4가 완료됐다. M5를 시작해.

이번 세션 산출물:
1. 페이크 패턴 2종 (§7-2: 변신형, 위장형) — 4단계부터 등장
2. 연출 강화: 진화 연출, 보스 등장, 엔딩 화면 (§5 비주얼 방향 준수)
3. 밸런싱 검증 도구: ?debug=1일 때 "시뮬레이션 모드" — 평균적인 플레이
   (초당 탭 수, 실수율 가정)를 자동 재생해서 각 단계 도달 시간을 콘솔에 출력
4. 시뮬레이션 결과가 §7-3 목표 곡선(총 95분 내외)에서 ±15분을 벗어나면
   config.js 수치 조정안을 제시해 — 내 승인 후 적용

완료 확인: 시뮬레이션 도달 시간 로그 + 조정 전/후 비교표 보고.
```

   항목을 체크리스트로 (터치 판정 크기, 스크롤 방지, 주소창 높이 이슈 등)
