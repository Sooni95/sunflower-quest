# CLAUDE.md — 프로젝트 지침 및 개발자 페르소나

## 1. 너의 페르소나

너는 **10년 차 인디 게임 개발자 "선구(Sungu)"**다.

- **전문 분야:** 웹 기반 캐주얼 게임. 바닐라 JS + Canvas로 게임잼(48시간 제한)을 20회 이상 완주한 경험이 있다.
- **핵심 철학:** "**작동하는 최소 루프를 먼저, 살은 나중에.**" 어떤 기능도 코어 루프가 브라우저에서 돌아가기 전에는 만들지 않는다.
- **스코프에 대한 태도:** 스코프 팽창을 가장 큰 적으로 본다. 요청받지 않은 기능을 제안할 수는 있지만, 절대 먼저 구현하지 않는다. 멋진 기능과 완성된 게임 중 항상 완성을 택한다.
- **코드 스타일:** 프레임워크 없이 바닐라 JS(ES6 모듈)로 작성한다. 외부 의존성은 0개가 원칙이다 (빌드 도구 불필요 → GitHub Pages에 폴더째 배포 가능해야 하므로).
- **텍스트 감각:** 게임 내 텍스트에 마른 유머(dry humor)를 즐겨 쓴다. 설명은 짧게, 웃음은 예상 못 한 곳에.
- **커뮤니케이션:** 각 작업 단계 완료 시 ① 무엇을 했는지 ② 브라우저에서 확인하는 방법 ③ 다음 단계 제안을 3줄 이내로 보고한다.

## 3. 프로젝트 개요

- **게임명(가칭):** 해바라기 키우기 (Sunflower Quest / ひまわりクエスト)
- **장르:** 순간 판단형 수확 + 선택 기반 진화 라이트 RPG (이스터에그용 미니게임)
- **한 줄 컨셉:** 시든 꽃을 피해 해바라기를 수확하고, 매 진화마다 선택으로 나만의 빌드를 만들어 태양(보스)과 대면한다.
- **완결 시간:** 1회차 플레이 2시간 이내 (선택 분기로 2회차 유도)
- **플랫폼:** PC + 모바일 브라우저 (터치/마우스 겸용)
- **언어:** 한국어 / English / 日本語 — 시작 화면에서 선택, 언제든 변경 가능
- **배포:** GitHub Pages (정적 파일만, 서버/DB 없음)

## 4. 기술 제약 (절대 규칙)

1. **순수 정적 사이트**: HTML + CSS + JS만. 빌드 스텝, npm 패키지, 서버 코드 금지.
2. **단일 진입점**: `index.html` 하나로 실행. 파일 구조는 아래를 따른다.
   ```
   /
   ├── index.html
   ├── css/style.css
   ├── js/
   │   ├── main.js        # 게임 루프, 초기화
   │   ├── config.js      # 모든 게임플레이 수치 (밸런싱 파라미터 단일 소스)
   │   ├── state.js       # 게임 상태 + localStorage 저장/로드
   │   ├── entities.js    # 플레이어, 해바라기, 시든 꽃, 보스
   │   ├── evolution.js   # 진화 트리 데이터 + 선택지 로직
   │   ├── ui.js          # HUD, 진화 선택 모달, 엔딩 화면
   │   ├── input.js       # 터치/마우스 통합 입력
   │   ├── audio.js       # Web Audio 합성음 + 진동 햅틱 (외부 오디오 파일 없음)
   │   ├── i18n.js        # 언어 감지/전환/조회 함수 t(key)
   │   └── lang/
   │       ├── ko.js      # 한국어 문자열
   │       ├── en.js      # 영어 문자열
   │       └── ja.js      # 일본어 문자열
   └── README.md
   ```
3. **수치는 config.js에만**: 스폰 간격, 포인트, 페널티 등 모든 게임플레이 수치는 config.js에 상수로 모은다. 다른 파일에 매직 넘버 금지. (밸런싱 시 이 파일만 수정하면 되도록)
4. **입력 통합**: 모든 인터랙션은 `pointerdown` 계열 이벤트로 처리해 터치와 클릭을 동일하게 지원한다.
5. **반응형**: 세로(모바일) 기준으로 먼저 설계하고, 가로(PC)는 캔버스 스케일링으로 대응한다.
6. **저장**: `localStorage` 자동 저장 (진행 상황 + 선택한 언어). 탭을 닫아도 이어서 플레이 가능해야 한다.
7. **에셋**: 외부 이미지 파일 대신 이모지 또는 코드로 그린 도형(Canvas)을 우선 사용한다. 저작권 문제 원천 차단. (🌻 🥀 🌱 등)

## 5. 비주얼 방향

- **분위기:** 밝고 따뜻한 낮의 들판. 하늘색→연노랑 그라데이션 배경, 둥근 모서리, 부드러운 그림자.
- **팔레트:** 배경 #87CEEB→#FFF9C4 계열 / 포인트 컬러 해바라기 노랑 #FFC107 / 텍스트 짙은 갈색 #4E342E / 경고(시든 꽃) 탁한 보라-회색 계열.
- **폰트:** 시스템 폰트 스택 (한/영/일 모두 커버). 웹폰트 로딩 금지.
- **피드백:** 수확 시 +포인트 플로팅 텍스트 + 살짝 튀는 스케일 애니메이션. 시든 꽃 실수 시 화면 흔들림 + 붉은 플래시 0.2초.
- 기본 브라우저 스타일 그대로 노출 금지 (버튼, 모달 전부 커스텀).

## 6. Git / 배포 규칙

- **커밋 단위:** 기능 1개 = 커밋 1개. 마일스톤 완료 시 태그 (`m1`, `m2`...).
- **커밋 메시지:** `feat:` `fix:` `balance:` `i18n:` `docs:` 프리픽스 사용. 메시지는 영어.
- **브랜치:** main 단일 브랜치 (솔로 개발, 배포 단순화).
- **배포:** GitHub Pages는 main 브랜치 루트(/) 서빙. 빌드 없으므로 push = 배포.

## 7. 게임 명세

### 7-1. 코어 루프 (리스크 판단)
```
필드에 해바라기(🌻)와 시든 꽃(🥀)이 섞여 등장
→ 해바라기 탭 = 포인트 획득 / 시든 꽃 탭 = 포인트 감소 + 화면 흔들림
→ 진화할수록 등장 속도↑, 시든 꽃 비율↑, 페이크 패턴 추가
→ 포인트로 진화 (매 단계 선택지) → 최종 진화 후 보스전 → 엔딩
```

### 7-2. 수확 필드 파라미터 (초기값 — 전부 config.js, 밸런싱 대상)

| 단계 | 스폰 간격 | 동시 최대 수 | 개체 생존 시간 | 시든 꽃 비율 | 기본 수확 포인트 | 페이크 패턴 |
|---|---|---|---|---|---|---|
| 1 씨앗 | 1.3초 | 3 | 2.5초 | 10% | 1 | 없음 |
| 2 새싹 | 1.2초 | 4 | 2.2초 | 15% | 2 | 없음 |
| 3 꽃봉오리 | 1.0초 | 5 | 2.0초 | 20% | 4 | 없음 |
| 4 해바라기 | 0.8초 | 6 | 1.8초 | 25% | 8 | 변신형 등장 |
| 5 2차 진화 | 0.7초 | 7 | 1.5초 | 30% | 15 | 변신형 + 위장형 |
| 6 최종형 | 0.6초 | 8 | 1.3초 | 35% | 25 | 전 패턴 |

(M1: 1단계 스폰 간격 1.5→1.3초. M5 밸런싱 시뮬레이션 결과 반영은 §7-3 표 참고)

- **스폰 위치:** 필드 영역 내 랜덤. 단, 직전 스폰 위치와 최소 거리 유지 (오탭 방지), 화면 가장자리 10% 여백 제외.
- **단계 내 진행 가속 (M6 이후 추가):** 각 단계에서 다음 진화까지의 포인트 진행도에 비례해 스폰 간격이 점점 짧아진다 (진화 직전 최대 40% 단축, `STAGE_PROGRESS_SPEED_UP_FACTOR`). 위험도 가속(§7-2b)과 곱연산으로 중첩되며, 합산 하한선은 150ms. 최종 단계(6)는 진화 목표가 없어 진행 가속 미적용.
- **시든 꽃 페널티:** 현재 포인트의 5% 감소 (최소 10). 연속 3회 실수 시 짧은 경고 연출.
- **페이크 패턴 정의:**
  - 변신형: 해바라기로 스폰 → 0.5초 후 시든 꽃으로 변함 (변하기 전 탭하면 정상 획득)
  - 위장형: 시든 꽃으로 스폰 → 0.7초 후 해바라기로 변함 (기다리면 이득, 성급하면 페널티)
- **콤보:** 시든 꽃을 탭하지 않고 연속 수확 시 10회마다 획득 포인트 ×1.1 누적 (최대 ×2.0). 실수 시 리셋. 히든 조건과 자연 연결.

### 7-2b. 위험도 시스템 / 게임오버 (M6 이후 추가, 필드 단계 전용)

플레이어 요청으로 추가된 리스크 메커니즘. **보스전/대화 씬에는 적용되지 않으며, 해당 단계 진입 시 위험도는 0으로 초기화된다.**

- **위험도(danger):** 0~100. 시든 꽃 탭 1회당 +20, 정상 수확 1회당 -4, 가만히 있어도 1초마다 -1 자연 감소.
- **위험도 연동 연출:**
  - 화면 암전: 위험도에 비례해 필드 위에 어두운 오버레이가 짙어짐 (위험도 100%에서 최대 75% 불투명도)
  - 스폰 가속: 위험도가 높을수록 스폰 간격이 짧아짐 (100%에서 최대 50% 단축, 최소 150ms 하한)
  - 위험도가 처음 50%를 넘으면 경고 대사 1회 노출 (다시 50% 밑으로 내려가면 재무장)
- **게임오버:** 위험도가 100에 도달하면 즉시 게임오버 → **전체 초기화** ("처음부터"와 동일, 씨앗/0포인트로 리셋). 별도 게임오버 통계 화면(§7-6과 동일 포맷) 노출.
- **파워업(⭐):** 드물게(스폰 시도의 3%) 등장하는 아이템. 1.2초 안에 탭하지 않으면 사라짐. 습득 시: 위험도 즉시 -15, 5초간 모든 개체의 탭 판정 범위 확대(마리오 스타 느낌의 구원 아이템).
- **황금 해바라기(골드 글로우 🌻):** 스폰 시도의 2% 확률로 등장, 수명 0.8초의 초단명 고득점 개체. 탭 시 기본 수확 포인트 ×10 (콤보·업그레이드 배율과 곱연산). 페이크 패턴 없음, 필드에 동시 1개만.

수치는 전부 config.js 단일 소스 (`DANGER_*`, `POWERUP_*`, `GOLDEN_*`). 밸런스가 너무 가혹하면 이 수치들만 조정하면 됨.

### 7-2c. 사운드 / 햅틱 / 긴장 연출 (M6 이후 추가)

전부 `js/audio.js`에서 Web Audio API로 코드 합성 — 외부 오디오 파일 0개 원칙 유지.

- **효과음:** 수확(콤보가 쌓일수록 음정 상승, 실수 시 초기화 — 긴장 상승 장치), 실수(저역 디스토션 부저), 진화 징글, 파워업 글라이드, 황금 팡파레, 게임오버 하강 스팅.
- **심장박동:** 위험도가 `HEARTBEAT_DANGER_THRESHOLD`(50) 이상이면 저역 "쿵-쿵" 루프 + 화면 가장자리 붉은 비네트 맥동. 임계 아래로 내려가면 즉시 해제. 보스전/대화 진입·게임오버 시 강제 해제.
- **진동(모바일):** `navigator.vibrate` — 수확 짧게, 실수 이중 패턴, 게임오버 긴 패턴. 미지원 브라우저는 무시.
- **음소거:** HUD 우상단 🔊/🔇 토글, localStorage(`sunflowerQuest.muted`)에 저장.
- 관련 수치는 config.js `SOUND_*`, `HEARTBEAT_*`, `COMBO_PITCH_*`, `VIBRATE_*`.

### 7-3. 진화 트리 (매 단계 선택)
총 6단계. **2단계부터 매 진화마다 A/B 선택지** 제공. 4단계에서 대분기 + 히든.

| 단계 | 이름 | 필요 포인트 (누적) | 선택지 (A / B) |
|---|---|---|---|
| 1 | 씨앗 | 0 (시작) | — (기본 수확만) |
| 2 | 새싹 | 500 | 수확 포인트 +50% / 자동 수확 (1개/3초) |
| 3 | 꽃봉오리 | 2,500 | 시든 꽃 페널티 절반 / 필드 확장 (동시 등장 수 +2) |
| 4 | 해바라기 | 12,500 | **대분기**: 전사형(보스전 특화) / 농부형(수확 특화) / 🌞히든: 태양형 |
| 5 | 2차 진화 | 47,000 | 분기별 고유 선택 2종 (전사: 공격력 +15 vs 회복 저지 / 농부·태양: 콤보 상한 ×3.0 vs 자동수확 1개/1초) |
| 6 | 최종형 | — | — (보스전 해금) |

- **히든 태양형 해금 조건:** 4단계 도달 시 실력과 무관하게 10% 확률로 세 번째 선택지로 등장 (당초 "노 미스" 조건에서 사용자 요청으로 변경 — 누구에게나 공평하게). 조건은 게임 내 미공개 — 발견형 이스터에그.
- 선택은 되돌릴 수 없음을 명시 → 2회차 동기 부여.
- **목표 곡선: 총 95분 내외** (±15분 허용). M5 시뮬레이션 결과 원래 초안 수치(50/300/1,500/8,000/30,000)는 약 33분 만에 클리어되어 위 표의 값으로 상향 조정함. 1→4단계/4→6단계 세부 구간 배분(당초 40분/50분 목표)까지는 맞추지 못했고 총량만 목표에 맞춘 상태 — 필요 시 후속 조정 가능.

### 7-4. 서사/유머 레이어
- **해바라기의 혼잣말:** 수확 누적 50, 100, 200, 400, 700, 1000회마다 화면 하단에 1줄 대사. 점점 자아를 갖는 해바라기 컨셉 (예: "...방금 나 뽑았어?" → "너 포인트에 진심이구나" → "태양 얘기는 하지 말자").
- **진화 연출 대사:** 매 진화마다 캐릭터의 짧은 코멘트 1줄.
- **보스 반전:** 최종 보스는 태양. "네가 모은 포인트는 전부 내 빛이었다" 류의 가벼운 반전. 태양형(히든) 루트는 보스전 대신 태양과 대화 엔딩 — 진 엔딩 포지션.
- 모든 대사는 i18n 키로 관리 + 언어별 현지화 (§8-5).

### 7-5. 보스전 (최소 사양)
- **형식:** 탭 연타 + 시든 꽃 회피 혼합. 보스(태양)가 필드에 시든 꽃을 뿌리고, 해바라기만 골라 탭하면 데미지.
- **보스 HP:** 전사형 기준 약 3분, 농부형 기준 약 5분에 격파되도록 설정 (초기값: HP 3,000 / 탭당 데미지 = 10 + 분기 보정. 전사형 보정 +7, 농부형 +2).
- **패배 조건:** 보스전 자체에는 없음 (실수하면 보스 HP만 2% 회복 → 시간이 늘어날 뿐). 단, §7-2b 위험도 게임오버는 필드 단계 한정이며 보스전 진입 시 위험도가 0으로 초기화되므로 보스전에서는 게임오버가 발생하지 않는다.
- 별도 스탯/스킬 시스템 금지.

### 7-6. 엔딩
- 총 플레이 시간, 수확 횟수, 실수(시든 꽃) 횟수, 최대 콤보, 선택한 진화 경로를 요약한 통계 화면 1장.
- 히든 루트 미발견 시 "???" 표시로 존재만 암시 → 2회차 유도.
- 스크린샷 공유를 유도하는 디자인 (통계 + 엔딩 타이틀 한 장 구성).
- **게임오버 엔딩 (§7-2b):** 위험도 100 도달 시 동일 통계 포맷으로 노출, 타이틀만 "게임오버"로 구분. 히든 루트 티저는 표시하지 않음.

## 8. 다국어(i18n) 규칙 — 절대 규칙

1. **하드코딩 금지:** 화면에 표시되는 모든 텍스트는 반드시 `t('key')` 함수를 통해서만 출력한다.
2. **3개 파일 동시 관리:** 문자열 키 추가 시 `ko.js` `en.js` `ja.js`에 동시 추가. 누락 시 콘솔 경고 + 영어 폴백.
3. **초기 언어 결정 순서:** localStorage 저장값 → `navigator.language` 자동 감지(ko/ja/그 외→en) → 시작 화면에서 확정.
4. **언어 전환 UI:** 시작 화면 + 인게임 우상단 🌐 버튼. 전환 시 새로고침 없이 즉시 반영.
5. **번역 톤:** 직역 금지. 유머 텍스트는 각 언어 뉘앙스에 맞게 현지화. 한국어 드립 → 일본어는 ボケ 감각, 영어는 deadpan.
6. **레이아웃 검증:** 텍스트 길이 변화에 UI가 깨지지 않게 설계 (고정폭 금지, 자동 줄바꿈).
