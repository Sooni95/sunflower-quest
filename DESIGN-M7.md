# DESIGN-M7 — 필드 이벤트 & 신규 개체 (루프 다양화)

> 이 문서는 M7 착수 기획서다. 구현 세션은 **CLAUDE.md를 먼저 읽고** 이 문서를 따를 것.
> 문제 정의: 개체가 전부 제자리 + 매 순간 할 일이 동일 → "해바라기만 계속 잡는" 단조로움.
> 해법: 짧은 랜덤 이벤트로 리듬을 흔들고, 행동이 다른 개체를 섞는다.

## 우선순위

| 순위 | 항목 | 기대 효과 | 난이도 |
|---|---|---|---|
| P1 | A. 필드 이벤트 3종 | 40~90초마다 게임의 규칙이 잠깐 바뀜 — 단조로움 직접 타격 | 중 |
| P2 | B. 신규 개체 2종 (벌/씨앗주머니) | 탭 판단이 "해바라기냐 아니냐" 이상으로 다층화 | 중 |
| P3 | C. 순간 미션 | 단기 목표로 몰입 리듬 생성 | 하 |

P1만 들어가도 체감이 크게 변한다. P1 → P2 → P3 순서로 마일스톤 쪼개서 진행하고,
**각 P 완료 시 브라우저 검증 + 시뮬레이션 확인 후 다음으로** (한 번에 다 넣지 말 것).

## A. 필드 이벤트 (P1)

공통 규칙:
- 발동 조건: 2단계부터, 필드 페이즈 전용 (보스전/대화/모달 중 발생 금지. 진행 중 모달 열리면 이벤트 즉시 종료)
- 주기: 마지막 이벤트 종료 후 40~90초 사이 랜덤 (`EVENT_INTERVAL_MIN/MAX_MS`)
- 예고: 발동 1초 전 화면 상단 배너 + 경고음 (i18n 키 `event.<name>.incoming`)
- 지속: 이벤트별 5~8초 (`EVENT_<NAME>_DURATION_MS`)
- 동시 발생 금지, 종류는 가중치 랜덤 (`EVENT_WEIGHTS`)
- 이벤트 중 심장박동/위험도 시스템은 평소대로 동작

### A-1. 황금 러시 🌟 (보상 이벤트)
- 지속 5초. 모든 신규 스폰이 황금 해바라기(기존 ×10 배율 그대로), 스폰 간격 50% 단축
- 사운드: 밝은 아르페지오 루프 (audio.js에 이벤트 루프용 함수 추가)
- 주의: 포인트 인플레 요인 → 완료 후 시뮬레이션 재확인 필수 (아래 "밸런스 가드" 참조)

### A-2. 시든 폭풍 🥀 (위기 이벤트)
- 지속 7초. 시든 꽃 비율 80%로 급증 + 스폰 간격 30% 단축
- 노 미스로 버티면 종료 시 보너스: 위험도 -20 + 포인트 보너스 (`EVENT_STORM_CLEAR_BONUS` — 현 단계 basePoints × 20)
- 연출: 화면에 보라빛 틴트, 심장박동 강제 ON(위험도 무관), 종료 시 해제
- i18n: 버티기 성공 시 칭찬 대사 1줄 (`event.storm.cleared`)

### A-3. 돌풍 💨 (조작 이벤트)
- 지속 6초. 이벤트 동안 스폰되는 개체가 좌우로 떠다님 (사인파 이동, 진폭/속도 config)
- 이동 중 수확 포인트 ×1.5 (`EVENT_WIND_POINT_MULTIPLIER`)
- 구현 노트: 엔티티에 `motion` 필드 추가, main.js의 rAF 루프(cleanupExpired)에서 위치 갱신.
  DOM 위치 갱신은 transform 사용 (left/top 재계산보다 저렴)

## B. 신규 개체 (P2) — 4단계부터 등장

### B-1. 벌 🐝 (움직이는 회피물)
- 화면 한쪽 끝에서 반대쪽으로 천천히 가로질러 이동 후 퇴장 (수명 = 이동 시간)
- 탭하면: 포인트 페널티 없음, 대신 위험도 +10 + 콤보 리셋 (`BEE_DANGER_PENALTY`)
- 시든 꽃과의 차별점: "정지물 회피"가 아니라 "이동 경로를 피해서 탭"하는 압박
- 스폰 확률 `BEE_SPAWN_CHANCE` = 스폰 시도의 4%, 동시 1마리
- 이동 경로가 다른 개체와 겹칠 수 있음 — 오탭 유발이 의도된 디자인

### B-2. 씨앗 주머니 🌰 (투자형)
- 탭하면 1초 후 그 자리 주변(반경 config)에 해바라기 3개 동시 팝 (`SEEDBAG_BURST_COUNT`)
- 팝된 해바라기는 일반 수명의 60%로 빨리 사라짐 — 몰아서 잡는 손맛
- 방치하면 그냥 소멸 (수명 1.5초)
- 스폰 확률 `SEEDBAG_SPAWN_CHANCE` = 3%, 동시 1개
- 주의: 팝 위치는 필드 경계 클램프 + 기존 최소 거리 규칙 무시(뭉쳐 나오는 게 컨셉)

## C. 순간 미션 (P3, 선택)

- 3단계부터, 2~4분 간격 랜덤으로 토스트 미션: "N초 안에 M개 수확!" (N=8, M=6 초기값)
- 성공: 포인트 보너스(현 단계 basePoints × 15) + 짧은 팡파레 / 실패: 페널티 없음, 조용히 사라짐
- **금지 조항 준수**: 미션 로그/목록 UI 금지 (CLAUDE.md §10 퀘스트 시스템 금지). 화면에 동시 1개, 단발 토스트만
- A/B와 동시 발생 금지 (이벤트 스케줄러가 통합 관리)

## 공통 구현 규칙

1. 수치는 전부 config.js (`EVENT_*`, `BEE_*`, `SEEDBAG_*`, `MISSION_*`). 매직 넘버 금지
2. 모든 신규 텍스트는 i18n 3개 파일 동시 추가 (ko 먼저, en/ja는 §8-5 톤 규칙 — 직역 금지)
3. 이벤트 스케줄러는 main.js에 단일 모듈로 (이벤트+미션 통합 큐, 동시 발생 방지)
4. 이동 개체 위치 갱신은 기존 rAF 루프에 편승 (새 루프 만들지 말 것)
5. localStorage 저장 대상 아님 — 이벤트/미션은 휘발성 (새로고침하면 리셋, 저장 스키마 건드리지 말 것)
6. CLAUDE.md에 §7-2d(이벤트), §7-2e(신규 개체) 섹션 추가할 것

## 밸런스 가드 (필수)

- 황금 러시/미션 보너스는 포인트 인플레 요인. **P1 완료 시점에 ?debug=1 시뮬레이션 실행**,
  총 시간이 95±15분에서 벗어나면 진화 임계값 재보정
- **절벽 주의**: CLAUDE.md §7-2 표 하단 경고 참조 — basePoints 대비 임계값을 올릴 때
  %페널티가 수입을 추월하면 진행 불능. 시뮬레이션 전 구간 유한한지 반드시 확인
- 시뮬레이션 도구는 이벤트를 모델링하지 않음 — 이벤트 기대 보너스를 반영하려면
  시뮬레이터에 "이벤트 평균 기여 PPS" 항목을 추가하는 것을 권장 (선택)
